# Security Policy

> Security policy version: **v1.0.0**

## Supported versions

| Version | Supported |
|---|---|
| `0.2.x` | ✅ |
| `<0.2.0` | ❌ |

## Reporting a vulnerability

**Please do not open public GitHub issues for security vulnerabilities.**

Report vulnerabilities privately using one of these channels:

1. GitHub Security Advisory (preferred):
   - `Security` tab -> `Report a vulnerability`
2. Email: `security@gitvibedev.example` (replace with project mailbox)

Include:

- Affected version and environment
- Reproduction steps
- Proof of concept (if available)
- Severity assessment and impact
- Suggested mitigation (optional)

## Response targets

- Initial acknowledgment: within 3 business days
- Triage status update: within 7 business days
- Fix timeline: based on severity and release risk

## Disclosure process

- Coordinated disclosure after patch availability.
- Credits are included in release notes unless reporter requests anonymity.
